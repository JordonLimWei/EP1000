Purpose of making a website

The purpose of creating a website is to allow us to have a place to document any type of information, be it a posting letter, receipt for a item paid or a project you are working on.
In this module, the main objective is to fabricate a physical object and a way to document the process is through making a website.
Advantages:
  • Easy to learn and use
  • Easy accessible and updatable
  • Able to store text, images, video and audio

What is HTML

HTML, which stands for Hypertext Markup Language, is the standard language for creating Web pages. It describes the structure of the Web page and also has elements that instructs the browser on how to display the content.
The structure is contained in a ".html" file in text form while the content is made up of two main parts, the head and the body. The head is the title at which the page is defined and the body is where the main content lies such as headings and paragraphs of text. Both the head and body are tags and are defined in start and end tags. There are more tags to allow you to structure your Web page in anyway that you like which are included in the next section of learning HTML.

Version Control

Version control is an essential part to documentation as it is a great way to track and manage changes done to your document. And a software that I will be using to do so is by Git.

What is Git

Git is a open-source software that tracks changes made to any files and consists of two segments, GitHub and GitHub Pages. GitHub makes use of Git and is a provider of Internet hosting for software development and version control. GitHub Pages builds a website hosted from your repository from GitHub where any edits and changes can be easily done and reverted if necessary.

Use of GitHub Desktop for version control

After downloading GitHub Desktop and signed in with my GitHub account, I created a repository locally such that the files changed in my system will be connected to the repository on GitHub and automatically make the changes onto the hosted website.
While documenting about this, I would need to commit the changes made to this html file, which I will be using as a demonstration of how to git commit and git push.
With reference to the image below, you would be able to see the changes made on the left-hand side of the screen.
Add a title to the changes made, in this case, documenting version control, press commit to main to git commit. The changes made will be held in the local repository.
Finally, if that is all of the changes you would like to make and want to host on the website, press push origin to git push.
And there you have it, you have learnt how to do git commit and git push and the changes has been made to your hosted website!
In addition, if you would like to recover or undo changes made, you can go to your history and recover older files which will restore your files to that version.

HTML vs Markdown

For the finalisation of which text editing method I would like to select to further the development of this documentation website, I have chosen to go with HTML, but this is not to say that Markdown is a bad method to documentation as admittedly it is the more efficient and easy to use method out of the two. However, with the requirements I need, HTML just have an edge on Markdown.

Firstly, HTML gives me full control over everything, down to the finest detail. As I am required to include numerous images and information to be displayed, Markdown is not able to achieve it in my opinion.

Secondly, with the integration of CSS, my site will look more presentable and professional which as compared to Markdown, it is leaps and bounds better.

Finally, HTML provides more interactivity to my website with features such as, drop-down menu and back-to-top button, which makes my website much easier to navigate. Therefore, I decide to use HTML over Markdown.
